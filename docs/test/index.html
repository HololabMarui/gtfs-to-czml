STEP 1 — PDF時刻表をAIで中間CSVに変換する

ChatGPT等のAIに以下のプロンプトをコピーして、PDF時刻表と一緒に送信してください。AIが4つの中間CSVを作成します。

AI用プロンプト 
あなたはGTFS-JP作成補助AIです。
添付PDFのコミュニティバス時刻表を読み取り、GTFS-JP Makerに取り込める中間CSVを作成してください。

出力は必ず以下の4つのCSVブロックに分けてください。
1. routes.csv
2. stops.csv
3. timetable.csv
4. calendar.csv

ルール:
- 推測した値は notes 列に理由を書く
- 読み取れない値は空欄にする
- 時刻は HH:MM 形式に統一する
- 24時を超える場合は 25:10 のように表記する
- 停留所IDは S001, S002, S003... の形式にする
- route_idは R001, R002... の形式にする
- service_idは weekday, saturday, holiday, everyday などにする
- 便名が無い場合は 第1便, 第2便... とする
- 方面がある場合は direction_id 0/1 に分ける
- PDF内の注記、運休日、運行条件は calendar.csv の description または notes に残す
- CSV以外の説明文は出力しない

routes.csv の列:
route_id,route_short_name,route_long_name,direction_id,direction_name,route_color,notes

stops.csv の列:
stop_id,stop_name,stop_lat,stop_lon,stop_sequence,notes

timetable.csv の列:
route_id,direction_id,service_id,trip_name,notes,停留所ID列...

calendar.csv の列:
service_id,monday,tuesday,wednesday,thursday,friday,saturday,sunday,start_date,end_date,description,notes

このツールについて

GTFS-JP Maker は、PDF時刻表から CZML / GeoJSON を生成するまでのワークフローを支援するブラウザアプリです。すべての処理はブラウザ上で完結し、データをサーバーに送信しません。

STEP 1 — AIでCSV作成

  1. 「プロンプトをコピー」ボタンでAI用プロンプトをコピーします。
  2. ChatGPT等のAIチャットを開き、PDF時刻表とプロンプトを貼り付けて送信します。
  3. AIが4つのCSVブロック（routes / stops / timetable / calendar）を出力します。
  4. 各CSVをテキストエディタまたはスプレッドシートで保存します。
  5. 不明な値（緯度経度など）はAI出力後に手動で補完することを推奨します。

STEP 2 — CSV → GTFS-JP ZIP

  1. 4つのCSVファイル（routes / stops / timetable / calendar）をアップロードします。
  2. プレビューで内容を確認します（先頭20行が表示されます）。
  3. 検査結果を確認し、エラーがある場合はCSVを修正して再アップロードします。
  4. 事業者名を入力します（必須）。
  5. 「GTFS-JP ZIP を生成する」ボタンで生成します。
  6. 「このZIPをSTEP 3で使用する」で直接STEP 3に渡せます。

STEP 3 — GTFS-JP → CZML/GeoJSON

  1. GTFS-JP ZIPを選択します（STEP 2から自動引き継ぎも可能）。
  2. サービス日を選択します（時刻表が存在する日付）。
  3. 対象路線・停留所マーカーなどを設定します。
  4. 「変換する」で CZML と stops.geojson を生成します。

中間CSV フォーマット

ファイル必須列説明
routes.csv route_id, route_short_name, route_long_name 路線情報。direction_id は 0（往路）/ 1（復路）
stops.csv stop_id, stop_name, stop_lat, stop_lon 緯度経度が空欄の場合、shapes.txt は生成されません
timetable.csv route_id, service_id, trip_name + 停留所ID列 6列目以降が停留所IDになる列。時刻は HH:MM 形式
calendar.csv service_id, monday〜sunday, start_date, end_date start_date / end_date は YYYYMMDD 形式

検査エラーと対処法

エラー内容対処法
必須CSVが読み込まれていない4つのCSVをすべてアップロードしてください
必須列が存在しないCSVの列名を確認してください（テンプレートを参照）
stop_id が重複しているstops.csv の stop_id が一意になるよう修正してください
route_id が routes.csv に存在しないtimetable.csv の route_id と routes.csv の route_id を一致させてください
service_id が calendar.csv に存在しないtimetable.csv の service_id と calendar.csv の service_id を一致させてください
timetable.csv の停留所ID列が stops.csv に存在しないtimetable.csv の列名（S001等）と stops.csv の stop_id を一致させてください
時刻形式が不正時刻は HH:MM 形式（例: 08:00, 25:10）で入力してください
