GTFS-JP ツール

GTFS-JPから動的なCZMLやGeoJSONを生成するツールです。

アニメCZML生成

車両モデルを使ったアニメーションCZMLを生成します。

GTFS-JPの入手先: https://www.gtfs.jp/

1.0 が標準です。0.5 で小さく、2.0 で大きくできます。

生成コマンド

変換スクリプト: gtfsjp_to_czml.py をダウンロード

CZMLとGLBモデルを使ったアニメーション出力用のコマンドです。

GeoJSON生成

ルート線とバス停を1回のコマンドでまとめて出力します。

COMMON

共通設定

GTFS-JPの入手先: https://www.gtfs.jp/

未入力なら全 route_id を対象にします。

LineString

バスルート描画

ルート線GeoJSONの出力先と、線の色・幅・透明度を設定します。

未入力なら routes.txt の route_color を使用します。

Point / marker-*

バス停描画

バス停GeoJSONの出力先と、マーカー色・種類・サイズを設定します。

このタブで生成されるファイル

  • 左側設定: routes.geojson などのルート線GeoJSON
  • 右側設定: stops.geojson などのバス停GeoJSON

生成コマンド

変換スクリプト: gtfs_shapes_and_stops_to_geojson.py をダウンロード

1回の実行で、ルート線GeoJSONとバス停GeoJSONを同時に生成します。

ここにコマンドが表示されます。

簡単な操作マニュアル

1. アニメCZML生成

  1. GTFS-JP の ZIP ファイルのパスを入力します。
  2. GLB ファイルの URL またはパスを入力します。
  3. 展開先ディレクトリ、出力CZMLパス、GLBサイズを確認します。
  4. 「コマンド生成」を押してターミナルで実行します。

2. GeoJSON生成

  1. 共通設定で ZIP、展開先、route_id、fallback を設定します。
  2. 左側の「バスルート描画」でルート線GeoJSONの出力先と線の見た目を設定します。
  3. 右側の「バス停描画」でバス停GeoJSONの出力先とマーカーの見た目を設定します。
  4. 「コマンド生成」を押して実行すると、2つのGeoJSONが出力されます。

3. 使い分け

  • 車両アニメーション表示をしたい場合は「アニメCZML生成」
  • 地図上にルート線と停留所を重ねて表示したい場合は「GeoJSON生成」
